# RegCompassR 1.7.0 condition-pooled architecture

RegCompassR 1.7.0 uses one condition-pooled RNA+ATAC evidence workflow with two
selectable structural scoring modes:

```text
condition × cell type cells pooled without sample-based selection or weighting
→ SuperCell2 condition-level metacells
→ one Pando GRN per condition × cell type
→ complete-GPR core reactions
→ core-reaction subsystem + KEGG/Reactome + master-Rhea reaction expansion
→ local FASTCORE feasibility completion
→ shared union meta-module GEM or shared full GEM
→ RNA support modified by coefficient-weighted ATAC accessibility state
→ directional two-step COMPASS-like LP
→ within-condition reaction ranking
→ descriptive pairwise condition comparison when multiple conditions are present
```

## 1. Biological design and pooling

Metacell strata are defined exclusively by `condition × cell type`. `sample_col`
is optional and defaults to `NULL`. When supplied, its values are not used for
cell selection, downsampling, weighting, grouping, or graph construction.

For downstream functions that require a sample-like metadata field, RegCompassR
creates a collision-safe internal identifier derived only from the condition.
Legacy `sample_balance` and `sample_balance_seed` values are ignored with a
warning. Therefore the metacell result is invariant to relabeling or removing
biological-sample metadata, provided condition and cell-type labels are unchanged.

The workflow retains cell-to-metacell membership and explicit provenance that the
pooling scope is `condition_x_celltype`. Condition-pooled metacells are descriptive
pseudo-observations, not independent biological replicates. One condition is valid
for descriptive ranking; multiple conditions support descriptive priority
comparisons but not biological-sample-level significance inference.

## 2. Pando parameters versus per-metacell evidence

Pando coefficients are learned within each `condition × cell type` from joint
RNA+ATAC data. They describe fitted regulatory relationships and are not an
independent validation dataset.

For a retained TF–peak–gene edge `e`, the per-metacell scoring state uses peak
accessibility. Accessibility support is centered and robustly scaled across all
conditions within the same cell type, then bounded with `tanh`:

\[
z_{e,u}=\tanh\!\left(\frac{A_{e,u}-\operatorname{median}(A_e)}
{s_e}\right).
\]

For target gene `g`, signed Pando coefficients define normalized edge weights;
model R-squared supplies a reliability shrinkage factor `rho_g`:

\[
R_{g,u}=\rho_g\sum_{e\to g}
\frac{|\beta_e|}{\sum_{k\to g}|\beta_k|}
\operatorname{sign}(\beta_e)z_{e,u},
\qquad -1\le R_{g,u}\le 1.
\]

Metacell TF RNA is not multiplied into this modifier. Target-gene RNA enters the
per-metacell support once, while ATAC supplies the changing regulatory state.
Because coefficients are estimated from the same pooled data, they remain learned
parameters rather than independent evidence.

## 3. Multiome gene support

For gene `g` in metacell `u`, full-transcriptome-library-size logCPM is converted
to bounded RNA support:

\[
C^{RNA}_{g,u}=\frac{x_{g,u}}{x_{g,u}+h}.
\]

The regulatory state modifies support on the log-odds scale:

\[
C^{MO}_{g,u}=
\frac{C^{RNA}_{g,u}2^{\alpha R_{g,u}}}
{1-C^{RNA}_{g,u}+C^{RNA}_{g,u}2^{\alpha R_{g,u}}}.
\]

The transform is bounded and zero preserving. ATAC can raise or lower existing
RNA support but cannot create positive support when RNA support is zero.

## 4. GPR aggregation

A reaction is a core reaction only when at least one complete GPR isozyme group
is present in the GRN-derived gene module. A partially represented required
complex is recorded as a candidate but is not treated as core.

For a complex containing `n` required subunits, the canonical AND rule is the
normalized Boltzmann soft minimum:

\[
C_{complex}=-\tau\log\left(\frac1n\sum_{i=1}^{n}
\exp[-C_i/\tau]\right),\qquad \tau=0.20.
\]

Its derivative with respect to every subunit is positive:

\[
\frac{\partial C_{complex}}{\partial C_i}=
\frac{\exp(-C_i/\tau)}{\sum_j\exp(-C_j/\tau)}>0.
\]

Isozyme groups are additive and no gene-promiscuity correction is applied:

\[
E_{r,u}=\sum_{k\in isozyme(r)}C_{complex,k,u}.
\]

`E` is reaction-expression support, not a flux, probability, or kinetic capacity.

## 5. Biological meta-module construction

Annotation-defined biological expansion is ordered:

1. complete-GPR core reactions;
2. all reactions in each core reaction's annotated subsystem;
3. reactions sharing KEGG or Reactome reaction identifiers;
4. reactions sharing the same master Rhea identifier.

Step 4 adds only reactions sharing a master Rhea ID; it does not import every
subsystem containing those reactions. No reaction is included merely because it
shares a metabolite with an included reaction. There is no metabolite-neighbour,
one-hop, stoichiometric-adjacency, or currency-metabolite expansion interface.

Local FASTCORE is applied after annotation-defined membership. It may add reactions
required to make the target module flux-consistent. Those additions are labelled
separately as FASTCORE feasibility support and are not reclassified as biological
meta-module membership.

## 6. COMPASS-like reaction penalty

Pando is not added as a separate reaction-confidence term. For expression-linked
reactions, the sole LP cost is

\[
p_{r,u}=\frac{1}{1+\log_2(1+E_{r,u})}.
\]

Non-finite reaction expression is first set to zero. Unmeasured expression and
explicit zero expression therefore both receive cost 1, the strictest
expression-linked penalty. Missingness remains available as a diagnostic flag.
Only exchange, demand, sink, and artificial-support reactions receive fixed
structural costs. Transport and cofactor reactions with GPR expression evidence
remain governed by the integrated multiome expression cost. The obsolete
`P_conf`, Q95, independent reaction-confidence, and `penalty_weights` interfaces
are absent.

## 7. Structural scoring modes

### 7.1 Shared union meta-module GEM

With `model_mode = "meta_module_gem"`, all locally FASTCORE-completed
condition-specific meta-modules are deduplicated into one union GEM. All conditions
and metacells use that same union structure.

### 7.2 Shared full GEM

With `model_mode = "full_gem"`, the complete validated GEM is retained and reused
for all targets and metacells. The target set is still derived from complete-GPR
core reactions, so the biological discovery layer remains identical; only the
structural context used by the LP changes.

Both modes use one shared medium, common bounds, identical reaction penalties,
and the same target-flux fraction. Condition differences therefore arise from the
multiome penalty matrix rather than condition-specific network structures.

## 8. Directional two-step LP

For target reaction `r` and direction `d in {+1,-1}`, step 1 computes

\[
v^{max}_{r,d}=\max_v d\,v_r
\quad\text{subject to}\quad Sv=0,\;l\le v\le u.
\]

Step 2 computes the minimum evidence-discordance cost:

\[
P^*_{r,d,u}=\min_{v,t}\sum_j p_{j,u}t_j
\]

subject to

\[
Sv=0,\quad l\le v\le u,\quad -t_j\le v_j\le t_j,
\quad d\,v_r\ge \omega v^{max}_{r,d},\quad \omega=0.95.
\]

Because the structural model and medium are shared, `vmax` must be invariant across
metacells for the same reaction, direction and medium. The implementation fails
if that invariant is violated beyond numerical tolerance.

## 9. Ranking and comparison outputs

Raw minimum penalty is valid for comparing the same reaction-direction-medium
target between conditions. It is not directly suitable for ranking unrelated
reactions because the required near-maximal target flux differs by target.

For cross-reaction priority, RegCompassR uses the normalized cost

\[
\widetilde P_{r,d,u}=
\frac{P^*_{r,d,u}}{\omega v^{max}_{r,d}},
\]

implemented as `penalty / (omega * vmax)`. This is the minimum network-wide
evidence cost per unit required near-maximal target flux in the selected structural
model. It does not represent physical flux or kinetic efficiency.

Within each `condition × cell type`, the metacell median of
\(\widetilde P\) is calculated and support is reported as

\[
A_{r,d,c}=-\log(\operatorname{median}_u\widetilde P_{r,d,u}+\varepsilon).
\]

Reactions are ranked by ascending median normalized cost. For the same reaction,
direction and medium, normalizing by shared `vmax` does not alter the ordering
between conditions.

- With one condition, `reaction_ranking` is the primary result and
  `condition_contrast` is empty.
- With two or more conditions, every condition retains its own ranking and all
  condition pairs receive descriptive `delta_support` comparisons.

These ranks and contrasts are descriptive because condition-pooled metacells are
not independent biological replicates. Results from `meta_module_gem` and
`full_gem` are separate structural-context analyses and should not be merged into
one ranking.
